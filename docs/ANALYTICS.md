# 통계·리포트 설계 (2026-07-20 초안)

세는 통계가 아니라 **판단을 바꾸는 통계**를 만든다. 각 항목은
`무엇을 결정하려고 보는가 → 어떻게 계산하는가 → 지금 데이터로 되는가` 순으로 적었다.

## 지금 DB에 있는 것 (2026-07-20 실측)

| 데이터 | 건수 | 통계 활용 |
|---|---|---|
| `units` 설치일자 | **876/876 (100%)** | 노후도 분석 즉시 가능 |
| `units` 제조사 | **876/876 (100%)** | 제조사별 비교 즉시 가능 |
| `units` 종류·형식·적재량·속도·층수 | 대부분 | 세분화 축 |
| `sites.maintenance_cost` (월 계약금액) | **0건** | ⚠️ **수익성 분석 전체가 막힘** |
| `failures` | 10건 | 실사용 후 |
| `billings.cost` (청구액) | 2건 | 실사용 후 |
| `material_requests` | 9건 | 실사용 후 |

→ **A그룹(자산 분석)은 오늘 만들 수 있고, B~D그룹은 데이터가 쌓여야 의미가 생긴다.**
가장 큰 병목은 계약금액이다. 이게 없으면 "이 현장이 돈이 되는가"를 한 줄도 계산할 수 없다.

---

## A. 자산 리포트 — 지금 바로 가능

### A-1. 노후도별 고장률
- **결정**: 몇 년 넘은 승강기부터 계약 조건을 다르게 가져갈 것인가
- **계산**: 설치연차 구간(0-9 / 10-19 / 20-29 / 30+)별 `고장 건수 ÷ 대수 ÷ 기간`
- **읽는 법**: 30년 이상 구간의 대당 연간 고장이 10년 미만의 3배라면, 그 구간은
  **정액 계약(POG)으로 받으면 손해**라는 근거가 된다
- 필요: `units.install_date`(있음) + `failures.unit_id`(있음, 데이터 축적 필요)

### A-2. 제조사별 고장률·부품비
- **결정**: 특정 제조사 현장을 늘릴 것인가 줄일 것인가, 견적 단가를 다르게 할 것인가
- **계산**: 제조사별 대당 고장 건수, 대당 연간 부품비(`billings.cost` 합계 ÷ 대수)
- **읽는 법**: 부품 조달이 어렵거나 비싼 제조사가 드러나면 **계약 단가에 반영**
- 필요: `units.manufacturer`(있음) + billings 축적

### A-3. 노후 자산 지도
- **결정**: 어느 현장에 교체 영업을 걸 것인가
- **계산**: 25년 이상 + 최근 1년 고장 3건 이상 + 부품비 상위 → **교체 제안 후보 목록**
- **읽는 법**: 이게 곧 개보수 영업 리스트다

## B. 계약 판단 리포트 — 계약금액 입력이 선행

### B-1. 현장별 손익
- **결정**: 재계약할 것인가, 조건을 바꿀 것인가, 놓을 것인가
- **계산**: `월 계약금액 × 12` − (`부품비` + `외주비` + `추정 인건비`)
  - 인건비는 실제 원가가 없으므로 **출동 건수 × 평균 출동 원가**로 추정
- **읽는 법**: 마이너스 현장을 세 갈래로 나눈다
  1. **노후 때문에 마이너스** → 부품 교체를 조건으로 연장 (교체 매출로 회수)
  2. **계약금액이 낮아서 마이너스** → 인상 협상
  3. **거리·접근성 때문에 마이너스** → 해지 또는 인근 업체와 교환
- 필요: ⚠️ `sites.maintenance_cost` **0건 — 최우선 입력 필요**

### B-2. 계약 만료 대시보드
- **결정**: 어느 계약부터 협상에 들어갈 것인가
- **계산**: 만료 D-day 순 정렬 + 각 현장의 손익·고장률·노후도를 한 줄에
- **읽는 법**: 만료 3개월 전 현장을 손익 나쁜 순으로 보면 협상 우선순위가 나온다
- 필요: `sites.contract_end`(있음) + B-1

### B-3. 계약 종류별 수익성 (POG vs FM 등)
- **결정**: 어떤 계약 형태를 밀 것인가
- **계산**: `contract_type`별 평균 손익률·대당 고장 건수
- 필요: B-1

## C. 인사 리포트 — 실사용 3개월 후

### C-1. 기사별 활동 요약
- **결정**: 우수사원 선정, 업무량 재분배
- **계산**: 1인당 — 출동 건수 / 평균 도착 소요(`dispatched_at`→`arrival_time`) /
  평균 처리 소요(`arrival_time`→`complete_time`) / 자체점검 완료율 /
  당직 횟수 / 연차 사용률 / 재고장 발생률
- **읽는 법**: 건수만 보면 쉬운 현장 담당이 유리해진다. **처리 소요 + 재고장률**을
  같이 봐야 "빨리 그리고 제대로"가 드러난다
- **재고장률**: 처리 완료 후 30일 내 같은 호기에 다시 고장 → 진짜 품질 지표
- 필요: `failures` 타임스탬프(스키마 있음) + `self_checks` + `duty_schedules` + `leaves`

### C-2. 업무 부하 균형
- **결정**: 담당 현장 재배정
- **계산**: 기사별 담당 대수 / 월 출동 건수 / 이동 추정 / 당직 횟수 편차
- **읽는 법**: 특정 기사에게 고장 많은 현장이 몰려 있으면 이직·번아웃 위험 신호

### C-3. 근태·연차 현황
- **계산**: 출근 체크율, 연차 소진율(미소진 이월 부담), 당직 편중도
- **읽는 법**: 연차를 못 쓰는 사람이 있으면 인력이 부족하다는 뜻

## D. 운영 리포트

### D-1. 부품 소비 통계 → 재고 예측
- **결정**: 무엇을 상비로 들고 다닐 것인가
- **계산**: 부품별 교체 빈도 / 제조사·기종별 교체 부품 상위 / 계절성
- **읽는 법**: 상위 20% 부품이 교체의 80%를 차지하면 그것만 차량 상비

### D-2. 고장 유형·계절성
- **계산**: `error_code` 유형별 분포, 월별·시간대별 발생 곡선
- **읽는 법**: 여름 문닫힘 급증 같은 패턴 → **당직 인원 배치와 사전 점검 항목**에 반영

### D-3. 응답 속도 추이
- **계산**: 접수→배정, 배정→도착, 도착→완료 각 구간 중앙값의 월별 추이
- **읽는 법**: 계약서상 응답 시간 준수 여부를 숫자로 증명 (민원·재계약 협상 자료)

### D-4. 갇힘 사고 별도 집계
- **계산**: 갇힘 유형만 분리해 현장·기종별 빈도, 평균 구조 시간
- **읽는 법**: 안전 이슈라 별도 관리. 공단 보고·보험 협상에도 쓰인다

## 선행 작업 (통계보다 먼저 해야 할 것)

1. **`sites.maintenance_cost` 입력** — 0건. 손익 리포트 전체의 전제.
   현장 수가 많으니 엑셀 일괄 입력이 현실적.
2. **부품 매입 단가** — 지금은 청구액(매출)만 있고 원가가 없어 마진 계산 불가.
   부품 마스터에 단가 컬럼이 필요하다.
3. **출동 원가 기준값** — 인건비·차량비를 건당 얼마로 볼지 회사가 정해줘야
   현장 손익을 추정할 수 있다 (정확할 필요는 없고 일관되면 된다).
4. **실사용 3개월** — C·D그룹은 표본이 없으면 숫자가 튄다.

## 만드는 순서 제안

1. **A그룹** (오늘 가능) — 노후도·제조사 분석. 데이터가 이미 100% 있다.
2. **B-2 계약 만료 대시보드** — 계약금액 없이도 만료일·고장률만으로 절반은 쓸모 있다.
3. **계약금액 입력 후 B-1** — 여기서부터 진짜 의사결정 도구가 된다.
4. **C·D** — 실사용 데이터가 쌓인 뒤.
