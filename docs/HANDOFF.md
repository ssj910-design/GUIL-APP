## 💬 UX 확인 요청 — 출근/근태 (차호근 → 친구, 2026-07-21)

> 출근체크·근무종료·위치 기능을 만들면서 **차호근이 이해한 UX**가 맞는지 확인 부탁해.
> 각 항목에 **O(맞음) / X(아님) + 한 줄 코멘트**만 주면 돼. 자세한 설계·플로우는 `docs/ATTENDANCE.md`.

**⏰ 시간 기준**
1. **정규 퇴근시간을 17:30으로 잡음** — 이 시각이 지나면 근무종료 버튼이 또렷해지고, 오늘 당직/숙직인 사람은 배지가 `당직 중`/`숙직 중`으로 바뀜. **회사 실제 퇴근시각 맞아?** (다르면 숫자만 바꾸면 됨)
2. **출근은 지각 판정 없이 누른 시각 그대로 기록** — 출근 기준시각(예: 09:00)이나 '지각' 개념은 안 넣음. 맞아?

**🏁 근무 종료 (퇴근/당직/숙직)**
3. **근무 종료 시, 오늘 근무표에 본인이 당직/숙직이면 그 버튼이 자동으로 뜸** — 기사가 매번 "나 당직인가 숙직인가" 고를 필요 없이 근무표대로. 맞아?
4. **근무표에 당직/숙직 배정이 있어도 [퇴근] 옵션은 남겨둠** — 그날 사정상 못 서는 경우용. 이땐 status가 `퇴근`으로 남아 관리자가 근무표와 다름을 앎. 이게 맞아, 아니면 배정된 사람은 무조건 당직/숙직만 눌러야 해?
5. **★중요★ 숙직 서는 기사는 마감 버튼을 실제로 언제 눌러?** — 저녁 퇴근 전에 미리? 아니면 다음날 새벽/아침에? (자정을 넘겨서 누르면 날짜가 꼬이는 문제가 있어서, 이 답에 따라 처리 방향이 갈려. `docs/ATTENDANCE.md` 6장 '알려진 이슈 1번' 참고)

**📍 위치**
6. **위치는 상시 추적이 아니라 딱 몇 시점에만** — 출근 · 현장 도착 · 처리완료 · 근무종료. + 대기 중 앱을 켜두면 2시간마다 자동 1회. **이 정도가 적절해? 2시간 자동 갱신이 과한가?**
7. **위치 공유를 끈 기사도 출근은 됨** — 대신 '가까운 현장 우선 배정'에선 빠짐. 맞아?
8. **위치 권한을 아예 안 켜면(공유 ON 상태) 출근 버튼이 비활성** — 위치를 못/안 켜는 기사는 마이페이지에서 '위치 공유 OFF'로 돌리면 출근됨. 이 흐름 맞아? (아이폰은 권한 감지가 약해서 별도 이슈 있음 — `docs/ATTENDANCE.md` 알려진 이슈 2번)

**👀 관리자**
9. **위치 안 켠 기사 안내는 게시판(우리방, 전원 공개)이 아니라 본인 앱 모달로만** — 특정 기사를 전원 앞에서 콕 집지 않게. 맞아?
10. **근무 중인데 앱을 2시간+ 안 본 기사는 관리자에게만 빨간 경고** — 기사 화면엔 안 보임(감시가 아니라 이상 신호 확인용). 맞아?

## 🚀 배포 운영 방침 (2026-07-21 합의)

> **앞으로 차호근 쪽(+클로드)은 커밋·push만 하고, 운영 배포는 친구가 판단해서 반영합니다.**
> `vercel.json`의 `ignoreCommand` 때문에 일반 push는 배포가 스킵됨(한도 절약). 운영 반영은
> 커밋 메시지에 `[deploy]`를 넣거나 `npm run deploy`(빈 `[deploy]` 커밋)로만 트리거됩니다.
> 차호근 쪽에서는 이제 `[deploy]`를 붙이지 않으니, **반영 타이밍은 친구가 정하면 됩니다.**

**현재 배포 대기 중인 변경 (2026-07-21, 우리 쪽 작업 — 친구가 반영 판단):**
- 출근 UX 전반: 근무종료 버튼 17:30 후 강조, **숙직 버튼 신설**, 근무 종료를 **근무표(당직/숙직)대로 자동 표시**, 위치 대기 로딩 표시
- 위치 안내 문구 순화('추적'·'배정' 표현 제거), 위치 미설정 안내를 **본인 앱 모달**로(게시판 브로드캐스트 제거)
- **⚠️ 백엔드 버그 3건 수정** — ①대기중 자동 위치갱신 **무한 루프**(실사용 시작하면 근무 중 기사 폰에서 GPS·DB 폭주) ②`TODAY_STR` 자정 고착(하루 넘게 켜두면 다음날 출근 버튼 안 뜸) ③`feed_read_at` 미전송
  - **①번은 실사용 데이터가 쌓이기 전에 반영하는 걸 권장** (지금은 데이터 0건이라 잠복 상태)

## 🔴 대기 중

### 1. Vercel 환경변수 — 웹 푸시 알림 활성화 (2026-07-21, 차호근)

**상태:** 로컬(차호근 환경)에서는 발송·수신 확인 완료. 배포본은 키가 없어 동작 안 함.
푸시를 제외한 나머지 기능은 배포본에서 정상.

**해야 할 일** — `docs/SETUP-ENV.md` 3번 항목 참고:
1. 저장소에서 키 생성:
   ```bash
   node -e "console.log(require('web-push').generateVAPIDKeys())"
   ```
2. Vercel → guil-app → Settings → Environment Variables 에 3개 등록
   (Production·Preview·Development 모두 체크):
   - `NEXT_PUBLIC_VAPID_PUBLIC_KEY` = 위 publicKey
   - `VAPID_PRIVATE_KEY` = 위 privateKey
   - `VAPID_SUBJECT` = `mailto:운영자메일`
3. Deployments → 맨 위 배포 ⋯ → Redeploy

> 키는 새로 만들어도 무방 (현재 실제 구독자 0명). 단 **직원들이 알림을 켠 뒤에는 교체 금지** — 기존 구독이 전부 무효화됩니다.

**확인:** 폰(안드로이드는 크롬 그대로, 아이폰은 홈 화면 추가 후)에서 마이페이지 → "이 기기에서 알림 받기" 켜고, 김기사 계정으로 고장 배정/출동 거부 시 알림 도착.
